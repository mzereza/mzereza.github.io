The program lets the user input a string of up to 0x80 bytes twice, and the program displays it insecurely with printf. We also have a print_flag function that will show us the flag.

The exploitation strategy will be:

  1. Leak PIE with Format Strings.
  2. Overwrite the GOT entry of puts with the address of print_flag.
  3. When puts is called we’ll have the flag.

Phase 1: Leak

What is a Format String Vulnerability?

A format string vulnerability appears when functions like printf directly interpret user input as a format string (printf(user_input)), allowing the use of specifiers like %p, %x, %s or %n. This gives control over stack reading (leaking useful addresses to break PIE/ASLR) and even arbitrary memory writing through %n. In real exploitation, this allows building read/write primitives without needing an overflow, making it ideal for attacking structures like the GOT.

In format string vulnerability challenges I always like to make a function to leak several indices, and then choose which ones to use. The function looks like this:

from pwn import *
binary = "./chall"

elf = context.binary = ELF(binary, checksec=False)
p = process(binary)

def leak_fmt(start, end):
    payload = b"ABCDEFG"
    for i in range(start, end):
        payload += f".%{i}$p".encode()
    p.sendline(payload)
    p.recvline()
    data = p.recvline()[:-1]
    parse = data.split(b".")
    for idx, value in enumerate(parse):
        log.info(f"{idx+start-1}: ".encode() + value)

This function also helps me see the index where my input starts. Whenever we see 0x2e47464544434241 it means our input starts at that index. That value represents “ABCDEFG.” in Little Endian.

Leaking the first 10 values we see that the start of our input is at index 6.

Leaking the next 20 we see a PIE leak at index 25.

Phase 2: Overwrite the GOT

Global Offset Table (GOT)

The Global Offset Table (GOT) is a table in memory that stores the real addresses of external functions (like puts, printf, etc.) once they are resolved at runtime. When the program calls a libc function, it actually first accesses its GOT entry, which points to the correct address. If you can overwrite one of its entries (for example, puts), you can redirect execution toward any function, like print_flag.

The next step after leaking PIE is to overwrite the puts entry in the GOT. To keep things simple we’ll use pwntools’ fmtstr_payload function. This function takes as arguments: fmtstr_payload(input_idx, {addr_to_overwrite: value}, write_size="byte|short|int"). The write_size is used to make the payload use %n, %hn or %hhn, which changes the output size and varies its efficiency.
